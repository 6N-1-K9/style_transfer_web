# Диаграмма последовательности обучения новой модели

### 1. Блок "Подготовка датасетов"

* пользователь выбирает директории датасетов для домена A и домена B;
* графический интерфейс передает запрос модулю работы с данными;
* модуль работы с данными обращается к файловой системе для получения списка изображений;
* файловая система возвращает данные о найденных изображениях;
* модуль работы с данными формирует предварительный просмотр датасетов;
* графический интерфейс отображает пользователю предпросмотр изображений.

### 2. Блок "Запуск обучения"

* пользователь задаёт параметры обучения модели;
* пользователь инициирует запуск процесса обучения;
* графический интерфейс передает параметры проекта управляющему модулю приложения;
* управляющий модуль инициирует запуск процесса обучения;
* модуль обучения CycleGAN запрашивает подготовленные данные;
* модуль работы с данными загружает изображения из файловой системы;
* подготовленные данные передаются в модуль обучения.

### 3. Блок "loop Для каждой эпохи"

* модуль обучения инициирует вычисления через библиотеку PyTorch;
* выполняется шаг обучения нейросетевой модели;
* вычисляются функции потерь и обновляются параметры модели;
* модуль обучения сохраняет статистику обучения;
* при необходимости сохраняется контрольная точка обучения (checkpoint);
* модуль обучения передает текущий статус выполнения управляющему модулю;
* графический интерфейс обновляет информацию о ходе обучения.

### 4. Блок "opt Остановка обучения"

* пользователь инициирует остановку процесса обучения;
* графический интерфейс передает команду остановки управляющему модулю;
* управляющий модуль передает сигнал остановки модулю обучения;
* модуль обучения завершает текущую итерацию и прекращает выполнение обучения.

# Диаграмма последовательности продолжения обучения с checkpoint

### 1. Блок "Выбор checkpoint"

* пользователь выбирает файл контрольной точки обучения;
* графический интерфейс отправляет запрос к файловой системе для проверки доступности checkpoint;
* файловая система возвращает информацию о выбранном checkpoint;
* графический интерфейс отображает пользователю информацию о найденной контрольной точке.

### 2. Блок "Восстановление состояния обучения"

* пользователь запускает продолжение обучения;
* графический интерфейс передает путь к checkpoint управляющему модулю;
* управляющий модуль инициирует продолжение обучения;
* модуль обучения загружает сохранённое состояние модели;
* модуль обучения загружает состояние оптимизаторов;
* библиотека PyTorch восстанавливает вычислительное состояние модели.

### 3. Блок "loop Для каждой оставшейся эпохи"

* модуль обучения выполняет шаг обучения модели;
* обновляются параметры нейронной сети;
* вычисляется статистика обучения;
* сохраняются новые checkpoint;
* сохраняются обновлённые статистические данные;
* модуль обучения передает статус выполнения управляющему модулю;
* графический интерфейс обновляет журнал выполнения и прогресс обучения.

### 4. Блок "opt Остановка обучения"

* пользователь инициирует остановку процесса обучения;
* графический интерфейс передает команду остановки управляющему модулю;
* управляющий модуль передает сигнал остановки модулю обучения;
* модуль обучения завершает выполнение текущего шага и прекращает обучение.

# Диаграмма последовательности выполнения инференса

### 1. Блок "Подготовка инференса"

* пользователь выбирает обученную модель;
* пользователь выбирает входное изображение или директорию изображений;
* пользователь указывает директорию для сохранения результатов;
* пользователь запускает процесс инференса;
* графический интерфейс передает параметры инференса управляющему модулю приложения.

### 2. Блок "Загрузка модели и данных"

* управляющий модуль инициирует выполнение инференса;
* модуль инференса загружает обученную модель из файловой системы;
* модуль инференса запрашивает входные изображения;
* модуль работы с данными считывает изображения из файловой системы;
* подготовленные изображения передаются в модуль инференса;
* модуль инференса загружает модель в вычислительную среду PyTorch.

### 3. Блок "alt Один файл / Директория изображений"

**Вариант 1 — обработка одного изображения**

* модуль инференса выполняет перенос стиля для одного изображения;
* результат сохраняется в указанной директории.

**Вариант 2 — обработка директории изображений**

* выполняется цикл обработки всех изображений в директории;
* для каждого изображения выполняется перенос стиля;
* результаты сохраняются в указанной директории.

### 4. Блок "Отображение результата"

* модуль инференса передает результат выполнения управляющему модулю;
* управляющий модуль передает данные в графический интерфейс;
* графический интерфейс отображает пользователю результат переноса стиля.
