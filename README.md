# Style Transfer Web

## О проекте

Данный проект представляет собой настольное приложение для переноса визуального стиля с одного набора изображений на другие изображения с использованием архитектуры **CycleGAN**.  
Проект разработан в рамках дипломной работы и ориентирован на практическое применение методов глубокого обучения в задаче **непарного перевода изображения на изображение**.

Основная идея проекта — предоставить пользователю удобный инструмент, который позволяет:

- обучить CycleGAN-модель на двух несвязанных датасетах изображений (Domain A и Domain B);
- сохранить и переиспользовать обученные модели;
- применять перенос стиля к произвольным изображениям без необходимости писать код;
- работать как через графический интерфейс (GUI), так и через консоль (CLI).

Приложение рассчитано на пользователей с базовым пониманием машинного обучения, но не требует глубоких знаний PyTorch или GAN-архитектур.

## 1. Запуск проекта с нуля

Проект разрабатывался и тестировался на дистрибутиве **Arch Linux** с графической оболочкой **Gnome**. 
На других дистрибутивах Linux с другими DE, а также на Windows и macOS возможны дополнительные сложности при установке зависимостей.
При выводе графики библиотекой Pywebview используются фреймворки qt или gtk (определяются автоматически), поэтому и могут возникнуть проблемы на Windosw и MacOS.

Планируется создание версий для Windows и MacOS, чтобы пользователь мог безболезненно запустить данный проект.

### 1.1 Требования

- Python **3.9+**
- pip
- Git
- GPU с поддержкой CUDA (опционально, но крайне желательно)
- Установленные драйверы NVIDIA и CUDA Toolkit (при использовании GPU)

### 1.2 Клонирование репозитория

```bash
git clone https://github.com/6N-1-K9/style_transfer_web.git
cd <style_transfer_web>
```

### 1.3 Создание виртуального окружения

Рекомендуется использовать виртуальное окружение:

```bash
python -m venv venv
source venv/bin/activate
```

### 1.4 Установка зависимостей

```bash
pip install --upgrade pip
pip install -r requirements.txt
```

Если используется GPU, убедитесь, что версия PyTorch соответствует установленной версии CUDA.
При необходимости PyTorch можно установить вручную с официального сайта:
[https://pytorch.org](https://pytorch.org)

### 1.5 Запуск приложения

```bash
python app.py
```

После запуска будет предложено выбрать режим работы:

```
1 - Web UI (графический интерфейс)
2 - CLI (консольный режим)
```
Консольный режим является прототипом проекта с упрощенным функционалом. Данный режим использовался мной для запуска обучения на удаленном сервере.

### 2.2 Ядро CycleGAN (папка `styler/`)

#### Модели (`models.py`)

* **StyleGenerator** — генератор на основе ResNet-блоков
* **Discriminator** — PatchGAN-дискриминатор
* **ResidualBlock** — остаточный блок генератора

Архитектура соответствует классической реализации CycleGAN.

#### Датасеты (`datasets.py`)

Используется **unpaired** датасет:

* изображения из Domain A и Domain B не имеют соответствий;
* выборка осуществляется независимо.

Все изображения приводятся к размеру `image_size` и нормализуются в диапазон `[-1, 1]`.

#### Обучение (`trainer.py`)

Класс `CycleGANTrainer` отвечает за:

* инициализацию моделей;
* настройку оптимизаторов и планировщиков;
* расчёт лоссов:

  * adversarial loss
  * cycle consistency loss
  * identity loss
* сохранение чекпоинтов и генераторов;
* возобновление обучения (resume).

Структура выходного проекта при обучении:

```
project_dir/
├── checkpoints/
├── models/
├── samples/
└── stats/
```

#### Инференс (`inference.py`)

Позволяет:

* загрузить сохранённый генератор;
* применить перенос стиля к одному изображению или папке изображений;
* сохранить результат в указанную директорию.

Генераторы сохраняются вместе с мета-информацией об архитектуре, что позволяет выполнять инференс без ручного указания параметров сети.

---

### 2.3 Пользовательский интерфейс

#### Web UI (`webview_app.py` + `web/`)

GUI реализован с помощью **PyWebview**, что позволяет:

* использовать HTML/CSS/JS;
* вызывать Python-функции напрямую из JavaScript;
* запускать обучение в отдельном потоке без блокировки интерфейса.

JavaScript взаимодействует с Python через `pywebview.api`.

#### CLI режим

CLI использует то же ядро, что и GUI, и предназначен для:

* автоматизации;
* запуска обучения на сервере без графической оболочки;
* отладки.

## 4. Подготовка датасетов

Для корректной работы рекомендуется:

* использовать изображения схожей тематики;
* привести изображения к одинаковому соотношению сторон;
* удалить повреждённые файлы.

В проекте присутствует вспомогательный скрипт:

```bash
python resize.py --input <dir> --output <dir> --size 256
```

## 5. Ограничения и особенности

* CycleGAN требует значительных вычислительных ресурсов;
* обучение на CPU крайне медленное;
* качество результата напрямую зависит от объёма и качества датасета;
* GUI ориентирован на Linux-системы, на Windows/macOS возможны проблемы с PyWebview.
